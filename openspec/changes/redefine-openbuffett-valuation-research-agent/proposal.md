## Why

当前项目仍以 `SmartIPO` 的美股 IPO 研究助手自居，但用户已经明确把产品目标重新定义为 `OpenBuffett`：核心主干不再是 IPO 信息整理，而是以美股为范围的专业级估值研究，再以此衍生打新分析与市场数据辅助能力。现在必须重构这一定义，因为现有默认 prompt、TUI 启动流程、品牌露出与分析收尾动作都仍服务旧定位，继续在旧语义上迭代只会放大能力错配、品牌混乱与研究输出不一致。

## What Changes

- 将默认产品身份从 `SmartIPO` 全面重定义为 `OpenBuffett`，清理当前运行面、源码表面、测试与现行文档中的旧品牌露出，但不改历史归档文件。
- 将默认 agent 主能力改为“美股专业级估值分析”，要求围绕公司质量、财务质量、估值区间、同板块可比公司、市场状态与多周期视角给出结论先行的研究输出；多周期默认采用 `1个月 / 6个月 / 1年 / 3年 / 5年`，且短期视角必须补充货币/经济政策、地缘政治、广泛市场情绪与资金流向判断。
- 对信息面、宏观、政策、地缘、情绪、资金流、新闻与 IPO 进度，要求 agent 必须联网获取最新信息，并对来源可信度、情绪化叙事和未证实项做审慎辨别与审计式记录。
- 为非股票代码输入建立“名称解析 -> 候选 ticker 推断 -> 用户确认 -> 才能进入正式分析”的前置门禁，避免 agent 在标的不确定时直接开做估值结论。
- 把“打新分析”重构为估值分析之上的延伸工作流，并增加严格门禁：仅对正在开放申购或即将开放申购的美股新股适用；已上市公司不得继续按打新逻辑分析。
- 明确当前跨品类市场数据能力的可承诺边界：优先支持股票、ETF 及现有数据面可覆盖的历史 OHLC / 市值研究，不把暂未接入的期货、期权、广义衍生品或加密数据伪装成完整支持。
- 在 TUI 中增加 agent 启动即开场自介的主动欢迎流程，且开场文案使用独立常量维护，不把品牌与欢迎话术散落在界面代码里。
- 约束 agent 在一次完整研究结尾主动询问用户，是否要在当前工作路径生成一份中文 Markdown 研究报告，并明确报告应结论先行、证据分级、承认未知。
- **BREAKING**：默认 system prompt 的任务边界、输出结构、能力优先级与品牌语义将发生实质变化，旧的 IPO-first 交互预期不再成立。

## Capabilities

### New Capabilities
- `openbuffett-agent-identity`: 定义默认 agent 的品牌身份、能力优先级、欢迎话术常量边界与研究报告追问契约。
- `valuation-research-workflow`: 定义专业级估值分析的默认研究主链，包括事实分级、可比公司对照、`1个月 / 6个月 / 1年 / 3年 / 5年` 估值判断、短期宏观与资金面约束、联网最新信息约束、来源可信度分级、情绪化叙事剔除、风险揭示与输出结构。
- `ticker-resolution-confirmation`: 定义当用户提供公司全名、模糊名或疑似笔误时，系统如何主动推断候选 ticker 并在用户确认前阻止正式估值分析继续进行。
- `ipo-subscription-analysis-gate`: 定义打新分析必须依附于估值分析，并且只适用于未上市、可申购或即将申购的新股研究场景。
- `tui-proactive-opening-surface`: 定义 TUI 启动即显示 agent 主动开场自介与推荐提问的行为，而不是等待用户先发第一句消息。

### Modified Capabilities
- 无

## Impact

- 受影响代码：`src/agent.py`、`src/tui/app.py`、`src/__init__.py`、`src/tool/__init__.py`、`src/tool/basic_tools.py`、`README.md`、`pyproject.toml` 以及与品牌、prompt、启动流程相关的测试文件。
- 受影响系统面：默认 agent composition、TUI 首屏体验、研究工作流路由、报告收尾动作、品牌对外表述。
- 受影响测试：需要重写默认 prompt 断言、品牌断言、开场消息断言、状态文案断言，并补充 ticker 确认与打新门禁的行为验证。
- 风险边界：若只改 prompt 不改 TUI，无法满足“agent 先于用户开场”；若夸大跨品类数据能力，会让产品承诺超出当前工具合同；若不把旧 IPO-first 语义从默认行为中剥离，后续实现会持续混合两套相互冲突的产品定义。
