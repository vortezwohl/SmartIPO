## ADDED Requirements

### Requirement: TUI 提交必须按队列串行执行
本地工作台 MUST 把同一会话内的用户提交维护为 FIFO 队列。当前一轮尚未完成时，新提交 MUST 进入待处理队列，而不是取消当前执行中的轮次或与其并发执行。

#### Scenario: 新提交在上一轮完成前进入队列
- **WHEN** 用户在当前轮次仍在处理时再次提交新消息
- **THEN** 系统 MUST 保留当前执行中的轮次继续运行
- **AND** 新消息 MUST 进入待处理队列
- **AND** 新消息 MUST 在当前轮次自然完成或失败后才开始执行

#### Scenario: 当前轮次结束后自动启动下一条
- **WHEN** 正在执行的轮次完成或失败，且待处理队列中仍有消息
- **THEN** 系统 MUST 自动启动队首消息对应的下一轮处理
- **AND** 系统 MUST 保持后续消息的原始提交顺序不变

### Requirement: TUI 必须展示所有排队中的用户消息
本地工作台 MUST 在时间线中保留每一条已提交但尚未完成的用户消息，并为其展示可区分的队列状态。

#### Scenario: 多条排队消息同时可见
- **WHEN** 用户连续提交多条消息，而其中只有第一条正在处理
- **THEN** 时间线 MUST 同时显示所有这些用户消息
- **AND** 正在执行的消息 MUST 显示为处理中
- **AND** 尚未执行的消息 MUST 显示为排队中，并带有队列顺位信息

#### Scenario: 队列推进时状态同步更新
- **WHEN** 队首消息完成，后续排队消息前移
- **THEN** 新的队首消息 MUST 从排队中变为处理中
- **AND** 其余排队消息的顺位显示 MUST 同步更新

### Requirement: 新会话与晚到事件不得污染队列状态
本地工作台 MUST 在创建新会话时清空待处理队列和当前活跃轮次，并忽略属于旧轮次的晚到事件。

#### Scenario: 新会话清空待处理消息
- **WHEN** 用户触发新会话
- **THEN** 系统 MUST 清空当前时间线中的排队状态
- **AND** 系统 MUST 重置本地待处理队列

#### Scenario: 旧轮次事件在新会话后到达
- **WHEN** 旧轮次 worker 在线程中延迟回传事件，且该会话已被重置或已切换到其他 active turn
- **THEN** 系统 MUST 忽略这些不属于当前 active turn 的事件
- **AND** 当前会话的时间线 MUST NOT 被旧轮次事件覆盖或追加错误状态
