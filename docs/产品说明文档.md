## 1. 项目名称

**OpenBuffett**: *一个以巴菲特式研究纪律为核心、面向美股估值研究的本地自主智能体工作台*

---

## 2. 项目简介

OpenBuffett 是一个聚焦**美股公司研究与估值分析**的本地 AI 投研 workbench。它不是一个泛化的财经聊天机器人，也不是单纯把大模型套上一层金融话术的“投资问答壳”，而是一个明确围绕**标的确认、证据分级、估值推演、可比公司对照、多周期判断、风险审计**来设计的研究型产品。

项目当前的核心目标是：

1. 帮助用户用更接近机构研究员的方法研究公司；
2. 把“公司基本面 + 估值框架 + 数据证据 + 宏观与情绪约束”放进同一条研究链路；
3. 让普通投资者在本地环境中，也能获得相对严谨、可解释、可复核的研究输出。

从产品定位上看，OpenBuffett 的优先级非常明确：

- **第一优先级：美股估值研究**
- **第二优先级：未上市且可申购 / 即将申购的美股 IPO / 打新分析**
- **第三优先级：市场数据辅助查询与研究输入补充**

---

## 3. 解决的问题与应用场景

### 3.1 解决的问题

传统散户投研常见的问题不是“没有信息”，而是：

- 有信息，但缺少统一研究框架；
- 能看新闻，但很难持续阅读财报、招股书、SEC 披露；
- 会看估值倍数，但不会拆解“当前价格隐含了怎样的增长、利润率和回报率预期”；
- 知道可比公司重要，但很难系统对照业务质量、资本结构、叙事溢价与风险折价；
- 短期容易被情绪和叙事牵引，中长期又缺乏结构化研究输出。

OpenBuffett 试图解决的，正是**“散户可以接触到很多信息，但难以形成机构化研究闭环”**这个问题。

### 3.2 适用场景

当前最适合以下场景：

- 研究一只美股当前到底**偏贵还是偏便宜**
- 对一家企业做**业务、财务、估值、可比公司**的系统性分析
- 对热门成长股、平台型公司、周期股、硬件股做**不同估值框架**下的判断
- 在公司尚未上市、但已经进入可申购或即将申购窗口时，做**IPO / 打新分析**
- 拉取**历史 OHLC、历史市值、关键指标、财务报表、估值输入、宏观辅助变量**作为研究底稿
- 输出一份中文 Markdown 研究报告，便于继续修改、审阅或归档

## 4. 核心功能说明

### 4.1 标的确认与 ticker 门禁

当用户输入公司全名、简称、别名、模糊名称甚至拼写不完全正确时，OpenBuffett 不会直接偷跑估值结论，而是会先进入**ticker 候选推断与确认**流程。只有在标的被可靠确认后，才进入正式研究。

这一步是产品的重要防误判设计，避免把错误公司当成目标标的继续分析。

### 4.2 估值研究主链

OpenBuffett 的默认研究流程并不是“先套一个 PE 再给建议”，而是要求围绕以下问题组织研究：

- 公司卖什么、护城河是什么；
- 行业位置和竞争格局如何；
- 收入、利润、现金流、资产负债表中，哪些质量最好，哪些最值得警惕；
- 当前价格隐含了怎样的增长、利润率、资本回报与市场叙事预期；
- 这些隐含预期是否合理，是否已经透支未来。

### 4.3 可比公司深度对照

项目强调**可比公司必须是研究主链的一部分**，而不是装饰项。当前产品设计要求可比分析至少覆盖：

- 估值倍数
- 增长速度
- 利润率
- 现金流质量
- 资本结构
- 市场叙事
- 流动性折价 / 溢价原因

### 4.4 多周期判断

OpenBuffett 默认输出 5 个时间尺度的判断：

- 1 个月
- 6 个月
- 1 年
- 3 年
- 5 年

其中短周期会额外关注：

- 利率路径
- 宏观政策
- 市场情绪
- 资金流向
- 板块拥挤度
- 波动性代理变量

中长期则更回到：

- 商业质量
- 财务质量
- 资本回报
- 竞争格局
- 估值回归空间

### 4.5 IPO / 打新分析门禁

打新分析**只在目标属于未上市且正处于申购窗口 / 即将申购窗口时成立**。如果目标已经上市，系统应退回常规二级市场估值分析，而不是继续沿用 IPO 叙事框架。

### 4.6 市场数据辅助研究

在当前数据面支持下，系统可辅助拉取与研究相关的数据，包括但不限于：

- 历史股价 OHLC
- 历史市值
- 公司概况
- 财务报表
- 财务增长
- 关键指标与财务比率
- DCF / 估值相关输出
- 分析师预期与目标价一致预期
- 可比公司
- 内部人交易
- 高管与治理信息
- 业绩会纪要
- 美债利率、市场风险溢价、宏观指标
- IPO 日历、披露、招股书相关接口

### 4.7 本地研究报告输出

系统提示词中已明确约定：在完整研究结束后，OpenBuffett 会主动追问是否需要在当前工作路径下生成一份**中文 Markdown 研究报告**。

当前仓库中已经有实际样例：

- `docs/NVDA_Valuation_Report_20260704.md`
- `docs/AAOI_valuation_20260705.md`

这说明项目不是只有“对话回答”，而是已经具备**可归档研究文稿**的产出能力。

## 5. AI 使用方式与技术方案

### 5.1 AI 在本项目中的角色

AI 在 OpenBuffett 中不是辅助功能，而是**核心研究引擎**。它承担的不是简单改写或摘要，而是以下职责：

- 组织研究流程
- 决定何时先确认 ticker、何时进入正式研究
- 依据提示词合同执行研究纪律
- 结合工具调用获取结构化数据
- 对来源做等级区分
- 生成中文研究结论
- 输出带边界、带不确定性说明的分析结果

### 5.2 当前采用的 AI 能力

当前代码主路径中，已经明确使用或体现了以下 AI 能力：

- 大语言模型驱动的研究问答与报告生成
- Agent Loop / 智能体运行时
- Tool Calling / 工具调用
- 流式事件输出（thinking / tool / assistant / compress / system）
- 会话压缩与长上下文管理
- 基于系统提示词的研究纪律约束
- 基于来源等级的输出边界控制
- 本地工作台式的人机协作界面

### 5.3 当前未采用或未作为主路径能力的部分

截至当前代码版本，以下能力**不是当前主路径核心**：

- 模型微调
- 向量数据库 / RAG 知识库
- 多模态图像理解
- 浏览器级网页自动化抓取
- 多智能体编排
- 公网 SaaS 化部署
- 用户账户体系 / 多租户

### 5.4 技术结构概览

当前产品可以概括为 4 层：

1. **交互层**：基于 Textual 的本地 TUI workbench
2. **Agent 层**：默认 Agent 组装、system prompt、工具集合、会话压缩
3. **工具层**：文件系统工具、基础工具、FMP 市场数据工具
4. **数据层**：FMP 结构化数据 + 简单网页抓取补充 + 研究输出文档

## 7. 数据来源、数据处理方式与成本说明

### 7.1 当前主要数据来源

#### A. 结构化金融数据主源：Financial Modeling Prep (FMP)

OpenBuffett 当前最核心、最明确的数据来源是 **Financial Modeling Prep (FMP) Stable API**。当前代码中围绕 FMP 封装了一个专门的研究客户端，并进一步暴露为默认研究工具集。

覆盖能力包括：

- IPO 日历、披露、招股书
- SEC filings 搜索
- 公司概况、报价、流通股
- 历史股价、历史市值
- 三大财务报表与增长
- 关键指标、财务比率、企业价值、owner earnings
- DCF / levered DCF / custom DCF
- 卖方一致预期、目标价一致预期
- 可比公司
- 高管、薪酬、机构持仓摘要
- 内部人交易
- 业绩会纪要
- Treasury、市场风险溢价、宏观指标

#### B. 公开网页补充抓取

项目还提供了一个轻量 `web_fetch_page` 工具，用于抓取单个公开网页并提取标题与正文摘录。它适合做少量公开页面补充，但**不是浏览器自动化，也不是完整网页工作流引擎**。

#### C. 一级披露优先的研究口径

虽然当前结构化数据主源是 FMP，但系统提示词明确规定：研究输出应优先把 **SEC / 公司公告 / 官方披露** 视为 A 级证据。也就是说，产品的研究方法论是**一级披露优先**，只是当前自动化数据接入以 FMP 为主。

### 7.2 当前使用的数据处理方式

当前主路径的数据处理方式以**按需拉取、会话内分析、必要时输出 Markdown 报告**为主，不是离线大库训练模式。

当前可确认的数据处理特点包括：

- 按问题触发工具调用，而不是先全量预抓取
- 优先获取结构化 JSON 结果，再由 Agent 进行解释与对照
- 输出时强调“已核事实 / 基于事实的推断 / 未证实项”分层
- 不依赖本地数据库作为核心存储层
- 不以内置回测引擎、向量库或训练流水线为当前主路径
- 研究结果主要以对话结果和 Markdown 研究报告沉淀

### 7.3 数据时间范围与市场范围

当前主路径非常明确：

- **市场范围**：以**美股**为主
- **研究重点**：估值研究、IPO / 打新研究、行情辅助
- **历史数据能力**：受 FMP 套餐能力约束
- **非美股、期货、期权、加密、广义衍生品**：当前不是主打范围；若数据不充分，应明确暴露边界

### 7.4 当前实际使用的数据套餐成本

本项目当前实际使用的是 **FMP Starter**。

按 FMP 官方定价页面在 **2026-07-05** 的公开信息，Starter 套餐为：

- **$29.00 / 月（billed monthly）**
- **300 API Calls / Minute**
- **Up to 5 Years of Historical Data**
- **US Coverage**
- **Annual Fundamentals and Ratios**
- **Historical Stock Price Data**
- **Profile and Reference Data**
- **Financial Market News**
- **Crypto and Forex**
- Trailing 30 days bandwidth：**20GB**

这一定价和能力范围，会直接影响 OpenBuffett 当前版本的数据边界，尤其是：

- 非美国市场覆盖有限
- 历史深度有限
- 某些高级数据能力不如更高套餐完整

### 7.5 数据合规与披露声明

当前项目面向的是公开市场研究，不以抓取个人隐私数据、非公开数据或内幕信息为目标。项目展示层面也不要求公开用户私有数据集。

需要明确的是：

- FMP 数据的展示与再分发，受其官方服务条款约束
- FMP 官方定价页已明确说明：**显示或再分发 FMP 数据需要额外的数据展示与授权协议**
- 因此，OpenBuffett 当前更适合作为**本地研究工具**与**研究辅助工作台**，而不是直接把 FMP 全量数据重新打包对外分发的 SaaS 数据平台

截至当前版本，OpenBuffett 更接近一个**本地终端工作台**，而不是公网 Web APP。

## 9. 核心创新点

### 9.1 不是泛财经聊天，而是研究纪律内建

很多“AI 金融产品”本质上还是通用问答，只是换了一套财经语气。OpenBuffett 的差异在于：**研究纪律是硬编码进产品身份与 system prompt 合同中的**。

它要求：

- 先确认标的，再做研究
- 区分已核事实、推断、未证实项
- 优先一级披露
- 强制多周期判断
- 强制可比公司对照
- 对打新和已上市标的做门禁分流

### 9.2 本地 workbench，而不是一次性脚本

项目不是“跑一次脚本、吐一段文本”这种形式，而是把 Agent 运行态做成了一个可交互的 TUI 工作台。从代码可以看到，它具备：

- 状态 banner
- 时间线视图
- thinking / tool / assistant / compress 事件显示
- 输入框与会话生命周期管理
- 新建会话
- slash 命令补全
- 队列与刷新机制

这使它更接近一个真正的研究工作台，而不是简易 Agent。

### 9.3 数据工具面较完整，能支撑完整研究闭环

当前默认工具面并不小：

- **7 个 FileGlide 文件工具**
- **3 个基础工具**
- **41 个 FMP 研究工具**

合计 **51 个默认工具入口**。

这意味着它已经具备从“拉数据 -> 审核信息 -> 组织研究 -> 输出报告”的较完整闭环基础。

### 9.4 显式承认边界，而不是伪装万能

OpenBuffett 的另一个创新点，不是“看起来无所不能”，而是**把边界显式化**：

- 标的不确定时先停下来确认
- 数据不足时降低置信度
- 研究范围默认限于美股
- 非主覆盖市场和衍生品不假装精确覆盖
- IPO 分析只在 IPO 场景成立

这种边界意识本身，就是严肃投研产品的重要特征。

## 11. 风险提示、局限性与未来改进方向

### 11.1 当前风险与局限性

1. **数据源依赖单一主源**  
   当前结构化研究数据高度依赖 FMP。若 API 限额、字段口径、套餐能力或服务稳定性变化，都会直接影响研究体验。

2. **市场覆盖范围明确受限**  
   当前主路径明确偏向美股，暂时不能说是“全市场万能投研系统”。

3. **Starter 套餐本身有能力边界**  
   当前实际使用的是 FMP Starter，历史深度、市场覆盖、部分高级数据能力都有限。

4. **不是投资建议系统**  
   项目更像研究辅助工具，不应该被误解为可直接替代持牌投顾、券商研究或合规投研流程。

5. **无公网 SaaS 化交付**  
   当前版本更适合作为本地研究工作台演示，而不是面向广泛用户的在线产品。

### 11.2 未来改进方向

1. 增加更稳定的**研究报告导出工作流**
2. 补充**更高质量的自动化测试与回归验证**
3. 增加更多**一级披露直连能力**
4. 增加**更细粒度的数据审计与缓存层**
5. 增强**非美股或更广市场覆盖**
6. 视比赛或产品化需要，再考虑 Web 化与多用户化

## 12. 开源与第三方材料说明

本项目并非直接提交一个未经改造的开源模板，而是在多个开源组件基础上，围绕“美股估值研究工作台”这一明确目标，做了如下层面的原创整合与产品化设计：

- 明确的 OpenBuffett 产品身份、边界与研究纪律
- 以估值研究为中心的 system prompt 合同
- 本地 TUI 工作台交互层
- FMP 研究工具层封装
- 标的确认、IPO 门禁、多周期判断、证据等级等研究规则
- 中文研究报告导向的输出约束
- 品牌视觉（SVG wordmark）与中文产品描述

其中，我的多个 Python 组件为本人自研并开源维护的技术基础设施；第三方组件如 Textual、requests、SymPy、FMP API 等，则在保留原始边界和协议前提下，被集成进 OpenBuffett 的产品闭环中。

## 13. 建议评审优先关注的五个点

如果评委希望快速判断项目价值，建议优先看这 5 点：
